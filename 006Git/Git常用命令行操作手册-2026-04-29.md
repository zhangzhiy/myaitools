# Git 常用命令行操作手册

## 1. 这份文档解决什么问题

这份文档面向日常开发中的常见 Git 操作，重点覆盖以下场景：

- 拉取最新代码
- 提交并推送代码
- 创建、切换、本地删除分支
- 拉取远程新分支到本地
- 暂存当前修改（stash / 贮藏）
- 查看当前状态和提交记录
- 处理“切分支前还有改动”的常见情况

如果只是日常开发，先把这份文档里的命令用熟，已经能覆盖大部分需求。

---

## 2. 进入项目目录

先进入你的 Git 仓库目录：

```bash
cd /你的项目目录
```

例如：

```bash
cd /Users/zzy/AndroidStudioProjects/landz-power
```

---

## 3. 最常用的状态查看命令

### 查看当前工作区状态

```bash
git status
```

这个命令最常用，建议养成习惯：

- 看当前在哪个分支
- 看哪些文件被修改了
- 看哪些文件已暂存
- 看是否可以直接提交

### 查看当前分支名

```bash
git branch --show-current
```

### 查看本地和远程分支

```bash
git branch
git branch -r
git branch -a
```

- `git branch`：查看本地分支
- `git branch -r`：查看远程分支
- `git branch -a`：查看本地 + 远程所有分支

### 查看提交记录

```bash
git log --oneline --graph --decorate -10
```

适合快速看最近 10 条提交。

---

## 4. 拉取最新代码

### 先抓取远程最新信息

```bash
git fetch
```

`fetch` 只更新远程分支信息，不会直接改你当前文件。

### 拉取当前分支对应的远程代码

```bash
git pull
```

这个命令通常等价于：

```bash
git fetch
git merge
```

### 更稳妥的拉代码方式

日常更推荐先看状态，再拉：

```bash
git status
git fetch
git pull
```

### 如果当前有本地修改，不建议直接 pull

先看状态：

```bash
git status
```

如果有未提交修改，优先考虑下面几种方式之一：

1. 先提交

```bash
git add .
git commit -m "保存当前进度"
git pull
```

2. 先 stash

```bash
git stash
git pull
git stash pop
```

3. 如果你确认本地改动不需要，才考虑丢弃

这类操作有风险，不建议在不确认的情况下执行。

---

## 5. 提交代码

### 查看哪些文件改了

```bash
git status
```

### 暂存文件

暂存全部已跟踪和未跟踪改动：

```bash
git add .
```

只暂存某个文件：

```bash
git add app/src/main/java/com/example/Test.kt
```

### 提交

```bash
git commit -m "修复登录页按钮点击无响应问题"
```

提交说明建议写清楚“做了什么”，不要只写 `update`、`fix` 这种过于模糊的内容。

---

## 6. 推送代码

### 把当前分支推到远程

```bash
git push
```

前提是这个本地分支已经和远程分支建立了跟踪关系。

### 第一次推送新分支

```bash
git push -u origin 分支名
```

例如：

```bash
git push -u origin feature/login-optimize
```

`-u` 的作用是建立本地分支和远程分支的跟踪关系。之后再推送时，直接执行：

```bash
git push
```

就可以了。

---

## 7. 创建和切换分支

### 查看当前所有本地分支

```bash
git branch
```

### 切换到已有分支

```bash
git switch 分支名
```

例如：

```bash
git switch master
```

如果你的 Git 版本较老，也可以用：

```bash
git checkout 分支名
```

### 新建并切换到新分支

```bash
git switch -c 新分支名
```

例如：

```bash
git switch -c feature/order-detail
```

老写法是：

```bash
git checkout -b feature/order-detail
```

### 切回上一个分支

```bash
git switch -
```

这个命令非常适合两个分支来回切换。

---

## 8. 拉取远程新分支到本地

这是最常见的场景之一：远程有人新建了分支，但你本地还没有。

### 先更新远程分支信息

```bash
git fetch origin
```

### 查看远程是否有这个分支

```bash
git branch -r
```

例如你看到：

```bash
origin/feature/new-ui
```

### 把远程新分支切到本地

```bash
git switch -c feature/new-ui --track origin/feature/new-ui
```

或者：

```bash
git checkout -b feature/new-ui origin/feature/new-ui
```

这样会：

- 在本地创建 `feature/new-ui`
- 跟踪远程 `origin/feature/new-ui`
- 自动切换过去

### 更短的写法

如果本地没有该分支，部分 Git 版本可直接：

```bash
git switch feature/new-ui
```

Git 会自动尝试从远程创建本地跟踪分支。但为了更稳妥，还是推荐显式使用 `--track`。

## 9. stash（贮藏 / 暂存当前改动）

当你代码写到一半，突然要切分支、拉代码、修别的问题时，`stash` 非常有用。

### 把当前改动先藏起来

```bash
git stash
```

执行后，工作区会恢复到最近一次提交的状态。

### 带说明地 stash

```bash
git stash push -m "登录页开发到一半，临时去修线上 bug"
```

推荐加说明，后面更容易分辨。

### 查看 stash 列表

```bash
git stash list
```

例如：

```bash
stash@{0}: On feature/login: 登录页开发中
stash@{1}: On master: 临时保存
```

### 恢复最近一次 stash，并从列表中移除

```bash
git stash pop
```

### 恢复指定 stash

```bash
git stash apply stash@{1}
```

`apply` 会恢复，但不会从 stash 列表中删除。

### 删除某一条 stash

```bash
git stash drop stash@{1}
```

### 清空所有 stash

```bash
git stash clear
```

这个操作不可恢复，执行前要确认。

## 10. 切分支前如果有本地改动，怎么处理

### 场景 1：当前改动就是要保留的

先提交：

```bash
git add .
git commit -m "保存当前开发进度"
git switch 目标分支
```

### 场景 2：改动还没准备提交，但需要临时切走

先 stash：

```bash
git stash push -m "临时切分支前保存现场"
git switch 目标分支
```

回来后恢复：

```bash
git switch 原分支
git stash pop
```

### 场景 3：改动不需要了

如果只是丢弃某个文件的修改：

```bash
git restore 文件路径
```

如果要丢弃所有未暂存修改：

```bash
git restore .
```

这类命令会直接覆盖本地修改，要谨慎使用。

## 11. 删除分支

### 删除本地分支

```bash
git branch -d 分支名
```

如果 Git 提示该分支未合并，但你确认要删：

```bash
git branch -D 分支名
```

### 删除远程分支

```bash
git push origin --delete 分支名
```

例如：

```bash
git push origin --delete feature/login-optimize
```

## 12. 常用查看与对比命令

### 查看改动内容

```bash
git diff
```

### 查看已暂存但未提交的内容

```bash
git diff --cached
```

### 查看某个文件是谁最后修改的

```bash
git blame 文件路径
```

### 查看某个文件提交历史

```bash
git log --oneline -- 文件路径
```

## 13. 每日开发推荐操作流程

### 开始开发前

```bash
git status
git branch --show-current
git fetch
git pull
```

### 开发完成后提交并推送

```bash
git status
git add .
git commit -m "本次修改说明"
git push
```

### 临时切去处理别的任务

```bash
git stash push -m "临时保存当前现场"
git switch 其他分支
```

回来后：

```bash
git switch 原分支
git stash pop
```

### 拉取别人刚推上去的新分支

```bash
git fetch origin
git switch -c 分支名 --track origin/分支名
```

## 14. 一份够用的速查清单

```bash
# 看状态
git status

# 看当前分支
git branch --show-current

# 拉最新代码
git fetch
git pull

# 新建并切分支
git switch -c 新分支名

# 切到已有分支
git switch 分支名

# 切回上一个分支
git switch -

# 拉远程新分支到本地
git fetch origin
git switch -c 本地分支名 --track origin/远程分支名

# 暂存改动
git stash push -m "说明"

# 恢复改动
git stash pop

# 提交代码
git add .
git commit -m "修改说明"

# 第一次推送新分支
git push -u origin 分支名

# 后续推送
git push
```

## 15. 最后建议

日常开发里，最重要的不是“记住所有命令”，而是先养成下面几个习惯：

- 操作前先执行一次 `git status`
- 拉代码前先确认有没有本地改动
- 切分支前先决定是提交、stash，还是丢弃
- 新分支第一次推送用 `git push -u origin 分支名`
- 对不确定会不会覆盖代码的命令，先停一下再执行
