# 基于 landz-power 项目通过 worktree 实现多分支管理操作步骤

## 1. 这份文档解决什么问题

在日常开发中，经常会遇到以下场景：

- 当前目录正在开发一个需求，但又要临时修另一个 bug
- 不想频繁在同一个目录里切换分支
- 希望让 Codex、Android Studio、终端分别在不同分支下并行工作
- 想保留当前分支现场，同时单独开一个目录处理新任务

Git 的 `worktree` 就是解决这类问题的标准方式。

它的核心价值不是“多一个分支”，而是：

- 让同一个 Git 仓库同时在多个目录展开
- 每个目录可以对应不同分支
- 多个分支可以同时存在、同时打开、同时运行

对于 `landz-power` 这种 Android 项目，`worktree` 非常适合多任务并行开发。

---

## 2. worktree 是什么

可以先把一个 Git 仓库理解成两部分：

- `.git`：仓库元数据，保存提交历史、分支、索引等信息
- 工作树（working tree）：你实际看到和编辑的源码目录

普通情况下，一个仓库只有一个工作树，也就是你当前项目目录。

例如：

`/Users/zzy/AndroidStudioProjects/landz-power`

这个目录通常就是当前仓库的主工作树。

而 `git worktree` 的作用，是为同一个仓库再额外创建新的工作树目录，例如：

- `/Users/zzy/AndroidStudioProjects/landz-power` 对应 `master`
- `/Users/zzy/AndroidStudioProjects/landz-power-xsxm-1306` 对应 `codex/xsxm-1306`

这样你就拥有了两个目录、两个分支，并且它们可以同时使用。

---

## 3. 什么时候适合使用 worktree

### 适合使用的场景

- 当前 `master` 或主开发分支还要保留，不能直接切走
- 同时处理多个 bug 或需求
- 希望把每个任务放到独立目录和独立分支中
- 想让 Codex 在一个目录修 bug，同时你自己在另一个目录继续开发
- 不想因为频繁切分支导致构建产物、IDE 状态、临时改动被打乱

### 不太适合使用的场景

- 只是改一个很小的问题
- 当前目录已经是干净的独立任务分支
- 仓库非常小，来回切分支成本很低
- 你暂时不需要并行开发

---

## 4. 在 landz-power 项目中使用 worktree 的推荐方式

当前项目假设位于：

`/Users/zzy/AndroidStudioProjects/landz-power`

推荐目录规则：

- 原项目目录保留不动
- 每个新任务开一个新的 worktree 目录
- worktree 目录名建议带任务号或用途
- 分支名建议统一加前缀，便于识别

例如处理 `XSXM-1306` 时：

- 原目录：`/Users/zzy/AndroidStudioProjects/landz-power`
- 新目录：`/Users/zzy/AndroidStudioProjects/landz-power-xsxm-1306`
- 新分支：`codex/xsxm-1306`

## 5. 标准操作步骤

### 步骤 1：进入主项目目录

```bash
cd /Users/zzy/AndroidStudioProjects/landz-power
```

### 步骤 2：确认当前仓库状态

```bash
git status
git branch --show-current
```

建议先确保当前目录状态清晰：

- 知道当前所在分支
- 知道是否有未提交改动

如果当前目录里有重要未提交内容，先不要混着做新任务。

### 步骤 3：创建新的 worktree 和分支

如果是新任务，推荐直接在创建 worktree 时同时创建新分支：

```bash
git worktree add ../landz-power-xsxm-1306 -b codex/xsxm-1306
```

这条命令做了三件事：

- 在当前目录的上一级创建新目录 `../landz-power-xsxm-1306`
- 创建新分支 `codex/xsxm-1306`
- 让新目录直接检出到这个分支

### 步骤 4：查看当前所有 worktree

```bash
git worktree list
```

你会看到类似结果：

```text
/Users/zzy/AndroidStudioProjects/landz-power             abcdef1 [master]
/Users/zzy/AndroidStudioProjects/landz-power-xsxm-1306   1234567 [codex/xsxm-1306]
```

这表示一个仓库现在有两个并行目录。

### 步骤 5：进入新目录开发

```bash
cd /Users/zzy/AndroidStudioProjects/landz-power-xsxm-1306
git branch --show-current
```

确认分支已经是：

`codex/xsxm-1306`

### 步骤 6：在新目录里运行项目

如果你要在新分支里运行项目，就直接在这个 worktree 目录里启动即可。

例如：

- Android Studio 直接打开新目录
- 或者在终端进入这个新目录后执行项目的运行命令

这时原目录和新目录可以同时存在，不互相影响。

## 6. Android Studio 中如何使用

对于 `landz-power` 这种 Android 项目，最常见的做法是：

### 方式 1：分别打开两个目录

- 一个 Android Studio 窗口打开：`/Users/zzy/AndroidStudioProjects/landz-power`
- 另一个 Android Studio 窗口打开：`/Users/zzy/AndroidStudioProjects/landz-power-xsxm-1306`

这样你就等于同时打开了两个分支。

### 方式 2：只在新任务目录里打开 Android Studio

如果当前任务只需要处理新 bug，可以直接关闭原目录工程，只打开新的 worktree 目录。

### 建议

如果你希望：

- 原目录保持稳定
- 新任务完全隔离

那就优先用独立 Android Studio 窗口打开 worktree 目录。

## 7. 在 Codex 中如何配合使用

如果你希望 Codex 在新的 worktree 目录里开发，直接在新目录启动即可：

```bash
codex -C /Users/zzy/AndroidStudioProjects/landz-power-xsxm-1306
```

如果你还想让 Codex 同时访问其他项目，例如后端项目：

```bash
codex -C /Users/zzy/AndroidStudioProjects/landz-power-xsxm-1306 \
  --add-dir /Users/zzy/backend/landz-client-domain
```

这样可以实现：

- 主要工作区在 `landz-power-xsxm-1306`
- 同时可读写 `landz-client-domain`
- 适合前后端联动排查问题

## 8. 常用命令清单

### 查看所有工作树

```bash
git worktree list
```

### 新建 worktree 并新建分支

```bash
git worktree add ../landz-power-xsxm-1306 -b codex/xsxm-1306
```

### 用已有分支创建 worktree

```bash
git worktree add ../landz-power-feature some-existing-branch
```

### 删除 worktree

```bash
git worktree remove /Users/zzy/AndroidStudioProjects/landz-power-xsxm-1306
```

### 删除不再需要的分支

```bash
git branch -d codex/xsxm-1306
```

### 清理失效记录

如果目录被手动删掉、记录没同步清理，可以执行：

```bash
git worktree prune
```

## 9. 一个完整示例

下面给出一个从创建到结束的完整流程。

### 场景

你当前正在 `master` 上工作，但需要单独处理 bug：`XSXM-1306`

### 操作流程

#### 1）进入项目

```bash
cd /Users/zzy/AndroidStudioProjects/landz-power
```

#### 2）查看当前状态

```bash
git status
git branch --show-current
```

#### 3）创建新的 worktree 和分支

```bash
git worktree add ../landz-power-xsxm-1306 -b codex/xsxm-1306 master 
```
其中  landz-power-xsxm-1306 是项目的文件名
codex/xsxm-1306 是分支名
master  是 依赖 
#### 4）进入新目录

```bash
cd /Users/zzy/AndroidStudioProjects/landz-power-xsxm-1306
```

#### 5）在新目录中运行项目或启动 Codex

```bash
codex -C /Users/zzy/AndroidStudioProjects/landz-power-xsxm-1306
```

#### 6）修复完成后提交代码

```bash
git status

```

#### 7）如果需要推送分支

```bash
git push -u origin codex/xsxm-1306
```

先确认新目录已经没有未处理改动，再执行：

```bash

```

如果分支也不再需要：

```bash
git branch -d codex/xsxm-1306
```

## 10. 常见问题

### 问题 1：为什么同一个分支不能同时出现在两个 worktree 中？

因为 Git 要避免你在两个目录里同时修改同一分支，导致状态混乱。

所以一个分支通常只能被一个 worktree 检出。

### 问题 2：我开了 worktree，还能继续用原目录吗？

可以。

原目录和新目录可以同时存在、同时编辑、同时打开 IDE。

### 问题 3：我能同时运行两个目录里的项目吗？

可以，但要注意运行资源冲突，例如：

- 端口冲突
- 同一个模拟器或设备冲突
- 缓存目录冲突
- 构建输出资源竞争

### 问题 4：删除 worktree 会不会把仓库删坏？

只要按正确方式使用 `git worktree remove`，不会把主仓库删坏。

但不要粗暴手删目录后不清理记录，否则后面要再执行 `git worktree prune`。

### 问题 5：当前目录有未提交改动时，能不能直接创建 worktree？

通常可以，但不建议把一个新任务建立在一个混乱现场上。

更稳妥的方式是：

- 先提交或暂存当前改动
- 再创建新的 worktree

## 11. 推荐命名规范

### worktree 目录命名

建议格式：

- `landz-power-任务号`
- `landz-power-用途`

例如：

- `landz-power-xsxm-1306`
- `landz-power-hotfix-login`
- `landz-power-refactor-owner-detail`

### 分支命名

建议格式：

- `codex/xsxm-1306`
- `fix/xsxm-1306`
- `feature/xxx`
- `hotfix/xxx`

如果主要是让 Codex 处理任务，用 `codex/任务号` 会很直观。

---

## 12. 推荐实践总结

对于 `landz-power`，建议形成以下习惯：

- 原目录尽量保持主分支或稳定开发分支
- 每个独立任务用一个独立 worktree
- 每个 worktree 对应一个独立分支
- Android Studio 按目录打开，不在一个目录里频繁切分支
- Codex 尽量在具体任务目录中启动

一句话总结：

> worktree 不是为了“多建一个分支”，而是为了让同一个仓库可以在多个目录中并行开发多个分支，从而减少切分支带来的混乱和上下文丢失。
