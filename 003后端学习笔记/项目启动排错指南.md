# 项目启动排错指南

> 记录 `landz-client-domain` 项目从零开始搭建到成功运行所遇到的所有问题及解决方案。

---

## 一、环境准备

### 1.1 必要条件

| 条件 | 说明 |
|------|------|
| JDK 17 | 项目使用 `corretto-17`，路径：`~/Library/Java/JavaVirtualMachines/corretto-17.0.15` |
| Maven | 通过 `brew install maven` 安装，路径：`/opt/homebrew/bin/mvn` |
| IntelliJ IDEA | 用于开发和运行项目 |
| 公司网络/VPN | **必须**，项目依赖 Eureka → Config Server → 数据库等 dev 环境基础设施 |

### 1.2 Maven settings.xml

文件位置：`~/.m2/settings.xml`

关键配置项：

```xml
<!-- 本地仓库路径，必须是当前用户的目录 -->
<localRepository>/Users/zzy/.m2/repository</localRepository>

<!-- Nexus 私有仓库 -->
<profiles>
    <profile>
        <id>LandzNexus</id>
        <repositories>
            <repository>
                <id>LandzNexus</id>
                <url>https://nexus.lizihang.com/repository/public/</url>
            </repository>
        </repositories>
    </profile>
</profiles>

<activeProfiles>
    <activeProfile>LandzNexus</activeProfile>
</activeProfiles>
```

**常见坑**：`localRepository` 路径如果指向了别人的用户目录（如 `/Users/liubingkai/...`），会导致所有依赖下载失败。

## 二、Maven 依赖问题

### 2.1 问题：大量 "package xxx does not exist" 编译错误

**现象**：缺少 `io.swagger.v3`、`org.springframework.cloud.openfeign`、`org.springframework.web` 等包。

**原因**：Maven 依赖下载失败后，会在本地仓库创建 `.lastUpdated` 标记文件，后续刷新时 Maven 看到标记就跳过下载。

**解决**：

```bash
# 删除所有失败标记
find ~/.m2/repository -name "*.lastUpdated" -delete

# 然后在 IntelliJ 中 Reload All Maven Projects
```

### 2.2 问题：IntelliJ 的 Bundled Maven 无法下载 Nexus 依赖

**现象**：命令行 `mvn` 能下载，但 IntelliJ 显示 "transfer failed"。

**解决**：在 IntelliJ 中切换 Maven 版本：

1. `Settings → Build, Execution, Deployment → Build Tools → Maven`
2. **Maven home path** 改为：`/opt/homebrew/Cellar/maven/3.9.14/libexec`
3. **User settings file** 勾选 Override，填：`/Users/zzy/.m2/settings.xml`
4. **Local repository** 勾选 Override，填：`/Users/zzy/.m2/repository`

### 2.3 问题：ClassNotFoundException: org.postgresql.Driver

**现象**：编译通过，但运行时找不到 PostgreSQL 驱动。

**原因**：多模块项目的传递依赖未正确安装。依赖链为 `starter → service → infrastructure → postgresql`，如果中间模块没有 install 到本地仓库，传递依赖会丢失。

**解决**：

```bash
# 在项目根目录执行 install，把所有模块安装到本地仓库
cd ~/backend/landz-client-domain
JAVA_HOME=~/Library/Java/JavaVirtualMachines/corretto-17.0.15/Contents/Home \
  mvn clean install -DskipTests
```

然后在 IntelliJ 中 **Reload All Maven Projects**。

## 三、项目启动配置

### 3.1 设置 Active Profile

**必须**指定 `spring.profiles.active=dev`，否则报错：

```
Could not resolve placeholder 'spring.profiles.active'
```

**IntelliJ 配置方法**：

1. 点击运行按钮旁的下拉箭头 → **Edit Configurations**
2. 选择 Application 配置
3. 在 **Program arguments** 中填入：`--spring.profiles.active=dev`
4. 保存

### 3.2 创建日志目录

Config Server 会下发 logback 配置，要求写入 `/var/log/landz/`，本地没有该目录会报 WARN。

```bash
sudo mkdir -p /var/log/landz/landz-client-domain
sudo chown -R $(whoami):staff /var/log/landz
```

### 3.3 启动方式

**方式一：IntelliJ 启动**（推荐日常开发）

配置好 Program arguments 后，直接点绿色运行按钮。

**方式二：命令行启动**

```bash
cd ~/backend/landz-client-domain
JAVA_HOME=~/Library/Java/JavaVirtualMachines/corretto-17.0.15/Contents/Home \
  mvn spring-boot:run -pl landz-client-starter -DskipTests \
  -Dspring-boot.run.arguments="--spring.profiles.active=dev"
```

## 四、启动成功标志

在控制台日志中看到以下内容即表示启动成功：

```
Tomcat started on port 8509 (http) with context path '/client-domain'
Started Application in XX seconds
```

**搜索关键词**：`Tomcat started on port` 可快速找到端口号。

### 4.1 访问地址

| 地址 | 说明 |
|------|------|
| http://localhost:8509/client-domain/doc.html | Knife4j 接口文档（Swagger UI） |
| http://localhost:8509/client-domain/actuator/health | 健康检查 |
| http://localhost:8509/client-domain/xxx | 具体 API 接口 |

> 注意：端口号可能被 Config Server 覆盖（如变为 8601），以日志中显示的为准。

## 五、可忽略的 WARN/ERROR

以下日志**不影响正常使用**，无需处理：

| 日志关键词 | 原因 |
|-----------|------|
| `RabbitMQ PRECONDITION_FAILED - inequivalent arg 'type' for exchange` | dev 环境 exchange 类型与代码声明不一致，日志末尾有 `continuing...` |
| `Elasticsearch health check failed - SSLHandshakeException` | ES 使用了 HTTPS，Java 不信任其证书，仅影响健康检查 |
| `Not found @TableId annotation, Cannot use Mybatis-Plus 'xxById'` | Oracle 旧表没有主键注解，不影响业务 |
| `Bean 'xxx' is not eligible for getting processed by all BeanPostProcessors` | Spring 内部 Bean 初始化顺序提示，不影响功能 |

## 六、关闭服务

| 方式 | 操作 |
|------|------|
| **IntelliJ 启动的** | 点击控制台左侧的红色 **■ 停止按钮** |
| **命令行启动的** | 在运行的终端按 `Ctrl + C` |
| **通用方式（强制关闭）** | 终端执行：`kill $(lsof -i :8509 -t)` |

## 七、启动流程全景图

```
你的电脑
  │
  ├─① 读取 bootstrap.yml
  │     → 找到 Eureka 地址：eureka1.dev.com:8203
  │
  ├─② 连接 Eureka，发现 Config Server
  │     → Config Server 地址：10.10.128.41:8202
  │
  ├─③ 从 Config Server 拉取配置
  │     → 数据库连接（PostgreSQL + Oracle）
  │     → Redis、RabbitMQ、MongoDB 配置
  │     → Logback 日志配置
  │     → 端口号（可能覆盖 bootstrap.yml 中的 8509）
  │
  ├─④ 初始化数据源
  │     → oracle-read、oracle-write
  │     → postgresql-read、postgresql-write
  │
  ├─⑤ 初始化 MyBatis-Plus、Feign Client、MongoDB、RabbitMQ
  │
  ├─⑥ 启动 Tomcat
  │     → port 8509, context-path /client-domain
  │
  └─⑦ 注册到 Eureka
        → 状态 UP，其他微服务可通过 Feign 调用本服务
```

## 八、常见排查命令速查

```bash
# 检查端口是否被占用
lsof -i :8509

# 杀掉占用端口的进程
kill $(lsof -i :8509 -t)

# 测试 Nexus 仓库连通性
curl -s -o /dev/null -w "%{http_code}" "https://nexus.lizihang.com/repository/public/"

# 清理 Maven 失败标记
find ~/.m2/repository -name "*.lastUpdated" -delete

# 命令行编译（不运行）
JAVA_HOME=~/Library/Java/JavaVirtualMachines/corretto-17.0.15/Contents/Home \
  mvn clean compile -DskipTests

# 命令行安装到本地仓库
JAVA_HOME=~/Library/Java/JavaVirtualMachines/corretto-17.0.15/Contents/Home \
  mvn clean install -DskipTests

# 查看某个模块的依赖树
JAVA_HOME=~/Library/Java/JavaVirtualMachines/corretto-17.0.15/Contents/Home \
  mvn dependency:tree -pl landz-client-starter

# 检查 Maven 版本
mvn --version
```

## 九、重要路径汇总

| 路径 | 说明 |
|------|------|
| `~/.m2/settings.xml` | Maven 全局配置（仓库地址、认证） |
| `~/.m2/repository/` | Maven 本地仓库（下载的 jar 包） |
| `bootstrap.yml` | Spring Cloud 启动配置（Eureka、Config Server） |
| `/var/log/landz/landz-client-domain/` | 运行时日志目录 |
| `landz-client-starter/` | 项目启动入口模块 |
| `Application.java` | main 方法所在类 |
